# Art Bible Direction v0.5 — **Mythic Coastal JRPG Fantasy** (D8)

Direction document; the full Art Bible (typography, exact palettes, model sheets, VFX specs) is authored after approval. Stated in full below — no reliance on prior versions.

## 1. Definition (D8)
Fortified harbors, elemental coastlines, readable silhouettes, guardian-scale spectacle, painterly world spaces, luminous transformation moments, and tactical battle clarity — broad enough to hold classic-JRPG inspiration while keeping Harbor Guardians original.

## 2. Visual pillars (binding)
1. Fortified harbor grandeur — every power center reads as a city-port fortress. 2. Guardian majesty — legendary, intelligent, bonded animal spirits; never pet mascots. 3. Elemental readability — Fire/Frost/Storm/Earth/Radiant/Umbral readable at thumbnail scale. 4. Cinematic resonance — heightened dramatic framing for battle and guardian powers. 5. Original silhouettes — no copied external armor/wings/weapons/UI/characters. 6. Mobile clarity — detail collapses cleanly into readable iconography.

## 3. World & regions
Mystical coastlines, sacred harbors, weathered stone and living water; sky/water as the emotional register. Region palette targets: Fire = ember ambers over black basalt shoals · Frost = pale cyan fjords, white-gold ice light · Storm = slate greens, lightning-silver headlands · Earth = loam golds, deep river greens · Radiant = luminous shallows, pearl and dawn-gold · Umbral = violet-black deltas, bioluminescent teal accents. The player's harbor re-tints toward the bonded guardian's identity.

## 4. Guardian shape language
Raxa (Tiger/Striker): low, forward, muscular; motion-trail stripes; amber-gold combat flares — realistic anatomy, minimal symbolism, visible physical tension, stopwatch sigil; this is the quality bar for all 20. Tarin (Elephant/Bulwark): monumental, grounded, shield-like; stone-gold armor planes; wall glyphs; heavy wave-breaker arcs. Nova (Owl/Oracle): lifted, layered feathers; moonlit geometry; cyan-gold eye sigils; star-map wings; scouting-lens effects. The remaining 17 derive shape language from species + chassis + element in the Kit Sheets, under the same rules.

## 5. Guardian Surge visuals (D9)
Luminous resonance states built from original motifs only: guardian crest halos · animal-spirit armor accents · harbor sigil light patterns · elemental tide rings · region-tinted resonance light. Staging: dramatic camera beat → sigil ignition → readable power silhouette → strong impact frames. Guardian Surge is a guardian-bond battle state — never a permanent form, species identity, or external spirit-possession/transformation copy. Avoided: dragon-wing default motif, external humanoid armor silhouettes, external color arrangements, any copied transformation visual.

## 6. Battle staging & UI readability
Theatrical but legible: turn bar always visible; hit-timing prompts as shape + icon + text (never color- or audio-only); strong impact frames; reduced-effects mode preserves all mechanical information. UI chrome: dark + gold flat marks (no gradients/shadows on marks); tactical overlays prioritize readability over ornament. Harbor View legibly shows walls, workers, visible surplus yards (exposed wealth is physical), docks, sanctum, and the guardian living in the city.

## 7. Economy / reward visualization
The physical-harbor doctrine includes exposed Crowns: exposed Crown surplus renders as physical counting-yards / coin stacks / strongboxes outside the protected Vault (visibly at-risk wealth, consistent with exposed Provisions yards, Iron stacks, Aether leybanks); safe Vault Crowns read as the secured fortified interior, visually distinct from the exposed yards. In-world objects use the painterly world palette; UI framing uses the dark + gold flat chrome.

## 8. Harbor Manifest tab visual language (D11/D12)
The **Harbor Manifest** is an in-world dockmaster's manifest with visually distinct tabs so rewards and information are never confused:
- **Claim Ledger** — reward packages as sealed crates / manifest entries with a **"Claim"** action; shows contents and slot usage (n/5, n/20).
- **Story Claims** — sealed/illuminated protected entries signaling permanence; distinct treatment from ordinary packages.
- **System Inbox** — messages as parchment notices / dispatch slips with **"Read / Acknowledge / Archive / View Reward Package"** actions and **never a "Claim" action**; clearly informational, no crate/coin imagery, so it never reads as a reward hold (UX1).
- **Reports** — a filter within System Inbox (offline returns, raid aftermath, migration summaries).

**Distinct-identity rules:** rewards use crate/coin/seal iconography + "Claim"; messages use parchment/dispatch iconography + Read/Acknowledge/Archive/View. **Critical vs normal notices are readable without color-only dependency** — critical messages carry a distinct shape/icon + label (e.g., a marked seal + "Urgent" text), satisfying the accessibility doctrine (shape + icon + text, never color alone). A system-grant notice visually points to its Claim Ledger package but shows no spendable amount itself; a pending-reward (full-slot) state is shown as a held manifest entry awaiting the Reward Delivery Resolution Screen, never as lost.

## 9. Pending-reward & full-storage visualization
Full-storage states are presented as opportunity, never failure: a full Vault/Granary/Ironworks/Crystal shows a "spend / trade / expand" prompt; a pending reward (cap full) shows the Resolution Screen framing ("make room to claim"), consistent with the no-hidden-loss doctrine.

## 10. Production notes
Painterly world rendering. `brand.config` remains the sole source of brand tokens. Asset pipeline: staged HF image tools + owner curation. Every asset is logged in the asset provenance checklist (source, prompt/creator, license, date); a failed or missing provenance entry blocks the asset. Reference boards are pattern-labeled, never asset-labeled.

## 11. IP boundary checklist (run per asset and per naming pass)
☐ No external game names/terms in product strings (incl. transformation-class, timed-attack, and spirit-meter terms from referenced classics) ☐ No copied/closely-imitated armor, wing, weapon, or character silhouettes ☐ No copied color arrangements or character designs ☐ No external UI layout or battle-menu arrangement reproduced ☐ No logo/trade-dress resemblance ☐ No official screenshots/art/renders/textures in production assets ☐ Every borrowed convention carries ≥1 original twist ☐ Provenance recorded. Any failed box blocks the asset.
