# Alpha A4 — Windows Desktop Acceptance

Acceptance evidence for the minimal Windows desktop player interface of the
**Bounded First Playable Expedition Loop** (owner Alpha A4 authorization
2026-07-23, Option A; brief §7 "documented Windows desktop acceptance").

## What the desktop app is

A Tauri 2 shell (`src-tauri`) that boots one native Windows window over the
static frontend in [`src/ui/shell`](../../src/ui/shell), served under the strict
CSP `default-src 'self'` — no dev server, no framework, no bundler, no inline
script/style, no IPC. The window renders the **committed, sim-derived**
playthrough transcript (`src/ui/shell/playthrough.json`), which is generated by
`pnpm run ui:playthrough` from the real expedition sim and drift-guarded by a
test (a fresh generation must byte-match the committed file). The viewer shows,
for each starting Guardian and for a cancellation run, every command in the
Harbor → route-anchor-outpost → Harbor loop with the resulting Safe Storage,
unsafe Overflow, and cargo-aboard state.

## Manual acceptance (interactive Windows session)

The steps below launch the actual window. They require an interactive desktop
session with a display and the Tauri/WebView2 runtime (they cannot run in a
headless CI/build environment, so they are documented for the owner rather than
auto-executed):

1. `cd src-tauri && cargo run --locked` (or `pnpm run tauri dev`).
2. A native window titled **"Harbor Guardians — First Playable Expedition
   Loop"** opens (1024×640).
3. The header shows the content/route/outpost ids; run tabs list the three
   Guardian full-success runs plus the cancellation (OPS1) run.
4. Selecting a run shows each step's command, phase, storage bars (Safe /
   Exposed / Overflow), and cargo aboard — matching the sim exactly.
5. Close the window; no crash, no network access (offline-safe, CSP `'self'`).

## Boundary

This is a minimal Alpha viewer: it renders the deterministic loop but wires no
gameplay input (no interactive commands, no save-file browser). Interactive UI,
input remapping, accessibility certification, and packaging/installers are
FUTURE BUILD (A4 hard exclusions / later phases).
